# Contributing to Closecode Website

We love your input! We want to make contributing to the Closecode website as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

## Pull Requests

Pull requests are the best way to propose changes to the codebase. We actively welcome your pull requests:

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Development Setup

1. Fork and clone the repository
2. Install dependencies: `npm install`
3. Start the development server: `npm start`
4. Make your changes
5. Test your changes in multiple browsers
6. Submit a pull request

## Code Style

* Use semantic HTML5 elements
* Follow CSS BEM methodology where applicable
* Use modern JavaScript (ES6+)
* Maintain responsive design principles
* Follow accessibility best practices
* Use meaningful commit messages

### CSS Guidelines

* Use CSS custom properties (variables) for consistency
* Mobile-first responsive design
* Use relative units (rem, em, %) over fixed pixels where appropriate
* Follow the existing color scheme and typography

### JavaScript Guidelines

* Use modern ES6+ syntax
* Write clear, self-documenting code
* Add comments for complex logic
* Use meaningful variable and function names
* Handle errors gracefully

## Reporting Bugs

Report bugs using GitHub's [issues](https://github.com/closecode/website/issues).

**Great Bug Reports** tend to have:

- A quick summary and/or background
- Steps to reproduce
  - Be specific!
  - Give sample code if you can
- What you expected would happen
- What actually happens
- Notes (possibly including why you think this might be happening, or stuff you tried that didn't work)

## Feature Requests

We love feature requests! Before creating one, please:

1. Check if a similar request already exists
2. Be specific about the feature and its use case
3. Consider the impact on existing users
4. Provide mockups or examples if applicable

## Design Changes

For design-related changes:

1. Ensure changes align with the Closecode brand identity
2. Maintain accessibility standards
3. Test across different devices and browsers
4. Consider the impact on page performance
